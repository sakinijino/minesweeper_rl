## 当前状态（EXP-019 系列完成后）

**最佳基准点**：

| 棋盘 | 最优实验 | 步数 | eval | 备注 |
|------|---------|------|------|------|
| 5×5×3 | EXP-018 | 2M | **85.6%** | EV=0.84，后续迁移推荐起点 |
| 6×6×5 | EXP-019a | 8.6M | **77%** | 小幅 reward，充分收敛 |
| 7×7×7 | EXP-019b-continue | 13M | **51.6%** | 仍在收敛，entropy 未 plateau |
| 8×8×10 | EXP-019c | 8M | **~0%** | 硬瓶颈，所有变体均失败 |

**核心已知结论**：
- win/lose=±1.0 导致 EV 卡死（~0.43）；缩小至 ±0.1 量级恢复正常（0.84）
- 小幅 reward（EXP-018 配置）对 6×6 和 7×7 有显著改善，但 8×8×10 仍无法突破
- 8×8×10 瓶颈不是 reward 幅度、LR、网络容量、课程路径——根因是极稀疏的 win signal（随机完成率 ~17%）
- 7×7×7 需要 10M+ 步才能充分收敛，比 6×6（8M）慢很多，说明棋盘越大越需要训练量

**8×8×10 根本瓶颈分析**：
- 扫雷策略分为两类操作：(1) 确定性安全格（约束传播可 100% 推断）；(2) 概率推断格（需统计推理或随机猜）
- 当前 CNN 对两类操作一视同仁，必须从原始观测自学约束传播规则，样本效率极低
- 解决路径：(A) 注入约束传播信息（改观测）；(B) 提供示范（行为克隆）；(C) 改变难度梯度（矿量课程）

## 方向 A：攻克大棋盘（核心目标）

目标：在 8×8×10（标准初级扫雷）上达到有意义的胜率（>10%）。

**已排除（全部失败，不必重跑）**：
- win/lose=±1.0 下所有变体：from scratch / Conv 迁移 / 分阶段课程（5→6→7→8）均 0%
- 进度奖励（踩雷 bonus / 揭格乘数）：均导致 reward hacking，0%
- LR cosine 衰减：无效，稀疏奖励瓶颈与 LR 无关
- 小幅 reward（±0.1）+ 完整课程路径：8M 步仍 0-3%（EXP-019c）

**当前优先方向**：

- [x] **A6. 用 EXP-018 reward 重跑完整课程学习**（EXP-019a/b/c）：完成。
  - Stage 6×6×5（EXP-019a）：**77%** ✅（EXP-015a±1.0 为 65%，+12%）
  - Stage 7×7×7（EXP-019b）：5M 步 27%，续训至 13M 步 **51.6%** ✅（EXP-015b±1.0 为 16%，+35.6%）
  - Stage 8×8×10（EXP-019c）：**2-3%** ❌（8×8×10 仍是硬瓶颈，reward 幅度不是关键因素）
  - **结论**：小幅 reward 在 6×6 和 7×7 上显著改善，但 8×8×10 无法突破。

- [ ] **A7. 内在动机探索奖励（curiosity / RND）**：8×8×10 的 win signal 极稀疏（随机探索赢率 <1%），per-step reveal 奖励密度不足且引发 hacking。RND（Random Network Distillation）或 ICM（Intrinsic Curiosity Module）提供基于状态新颖性的密集信号，不依赖任务奖励，从根源解决探索问题。
  - 注意：扫雷每局是全新随机棋盘，"状态新颖性"在 episode 间没有意义（每局都新颖）；episode 内新颖性可能有效但效果有限
  - 实现代价较高：需自定义 env wrapper 或 callback
  - **优先级：Tier 3**，先验证 A10/A11 是否能突破

- [ ] **A8. 纯稀疏奖励（去掉 reveal=0.1）**：EXP-012/013 已证明 reveal 奖励引发 reward hacking。但在 8×8×10 上去掉 reveal 信号会让学习更难。可在 7×7×7 上先验证效果再决定。
  - **优先级：Tier 3**，已知风险大

- [x] **A9. 7×7×7 继续训练到真正 plateau**（EXP-022）：EXP-019b-continue 在 13M 步时 entropy 仍在下降（-0.53），尚未完全 plateau。再跑 7M 步（共 20M），确认真实天花板并作为更好的 8×8×10 迁移起点。
  - **优先级：Tier 1**，低代价，高信息价值

- [ ] **A10. 约束传播安全通道**（EXP-020）：向 CNN 观测添加第 3 个通道——确定性安全掩码（通过约束传播推断的 100% 安全格子）。
  - 核心理念：将"确定性安全格"的识别从 RL 问题（需要大量样本）转化为观测直接提供的信号，CNN 只需学会"当 channel_2=1 时优先点击"
  - 已通过 TDD 实现并测试（compute_safe_mask + obs_channels=3）
  - 在 6×6×5 上对比 EXP-019a（77%）快速验证效果
  - **优先级：Tier 1**，最高潜在收益

- [ ] **A11. 矿量课程（8×8 固定棋盘，地雷从少到多）**：固定棋盘 8×8，地雷数从 1 逐步增加到 10。
  - 优势：不需要权重迁移（CONTINUE_FROM 继承所有权重），难度梯度更平滑，8×8×1/2 随机完成率接近 100%，给 Value function 充分的初始学习信号
  - 风险：难度突变点（7→8→9→10 颗地雷附近）可能仍存在跳变
  - **优先级：Tier 2**，Tier 1 结果出来后评估执行

- [ ] **A12. 行为克隆预训练**：用 Minesweeper 约束求解器（只点确定性安全格，无法确定时随机）生成高质量 demonstration，作为 BC 预训练数据，再 RL 微调。
  - 优势：给 value function 有意义的初始化，打破早期稀疏问题；A10 的 compute_safe_mask 逻辑可复用为 solver 核心
  - 代价：需写 BC 训练循环（~200 行代码）
  - **优先级：Tier 2**

- [ ] **A13. 8×8×10 超长训练（50M+ 步）**：增加 n_envs 后的暴力探索，代价较高但是最后的保险选项。
  - **优先级：Tier 3**，仅在 A10/A11 失败后考虑

## 方向 B：提升训练效率

- [ ] **B1. 最优 checkpoint 自动选择**：EXP-007 的 1.75M 优于 2M final（87% vs 86%），训练完取 final 会损失 1-2%。在训练时加入 eval callback，每隔若干步评估并自动保存最优 checkpoint，对大棋盘长训练尤其重要（避免浪费后期偏移）。
  - **优先级：运维改善，不影响算法效果，暂缓**

- [ ] **B2. entropy 衰减**（ent_coef 从 0.01 cosine 降到 0.001）：7×7×7 晚期 entropy 仍高（-0.53）。主动退火可在 plateau 后收紧策略、减少无效探索。SB3 支持 `ent_coef = lambda progress: ...` 格式。
  - **优先级：Tier 2**，结合 A10 结果决定

- [x] **B3. 增加并行环境数（n_envs 16→64）**（EXP-021）：Modal T4 的 GPU 利用率在 n_envs=16 时估计不足 30%，扫雷环境极简，SubprocVecEnv 可线性扩展。预期 2-4x 吞吐量，近乎零风险的速度提升。
  - **优先级：Tier 1**，与 EXP-020 并行验证

## 方向 C：模型结构（低优先级）

EXP-008 已排除容量假设（features_dim 256 无收益），以下改进独立于容量问题：

- [ ] **C1. 分离 pi/vf 特征提取器**：当前 policy 和 value function 共享 CNN，策略梯度更新干扰 value 拟合（尤其在大棋盘稀疏场景下，EV 爬升缓慢可能部分源于此）。分离后 value head 可独立优化。
  - **优先级：Tier 3**
- [ ] **C2. 更深 pi/vf 全连接层**（[128,128]）：当前 [64,64]，对 8×8×10 的复杂局面决策可能不足。与大棋盘实验结合验证。
  - **优先级：Tier 3**
