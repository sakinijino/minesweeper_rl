## 方向 A：攻克大棋盘（核心目标）

目标：在 8×8×10（标准初级扫雷）上达到有意义的胜率（>10%）。

**已排除（全部失败，不必重跑）**：
- win/lose=±1.0 下所有变体：from scratch / Conv 迁移 / 分阶段课程（5→6→7→8）均 0%
- 进度奖励（踩雷 bonus / 揭格乘数）：均导致 reward hacking，0%
- LR cosine 衰减：无效，稀疏奖励瓶颈与 LR 无关
- 小幅 reward（±0.1）+ 完整课程路径：8M 步仍 0-3%（EXP-019c）

**当前优先方向**：

- [x] **A6. 用 EXP-018 reward 重跑完整课程学习**（EXP-019a/b/c）：完成。
  - Stage 6×6×5（EXP-019a）：**77%** ✅（EXP-015a±1.0 为 65%，+12%）
  - Stage 7×7×7（EXP-019b）：5M 步 27%，续训至 13M 步 **51.6%** ✅（EXP-015b±1.0 为 16%，+35.6%）
  - Stage 8×8×10（EXP-019c）：**2-3%** ❌（8×8×10 仍是硬瓶颈，reward 幅度不是关键因素）
  - **结论**：小幅 reward 在 6×6 和 7×7 上显著改善，但 8×8×10 无法突破。下一步需要 A7 或根本性架构改变。

- [ ] **A7. 内在动机探索奖励（curiosity / RND）**：8×8×10 的 win signal 极稀疏（随机探索赢率 <1%），per-step reveal 奖励密度不足且引发 hacking。RND（Random Network Distillation）或 ICM（Intrinsic Curiosity Module）提供基于状态新颖性的密集信号，不依赖任务奖励，从根源解决探索问题。
  - 实现代价较高：需自定义 env wrapper 或 callback
  - 可与课程学习结合：在稀疏阶段（7×7→8×8）开启 intrinsic reward
  - **推荐优先尝试**：是目前唯一未排除且理论上能解决稀疏问题的方向

- [ ] **A8. 纯稀疏奖励（去掉 reveal=0.1）**：当前 reveal=0.1 是主要 reward hacking 的来源（每步揭格即得奖励，赢不赢无关）。只保留 win/lose 终止信号，彻底消除中间奖励的 hacking 面，policy 只能从最终结果学习。
  - 风险：信号更稀疏，探索更难；但 hacking 不可能发生
  - 与 A7 搭配：intrinsic reward 填补密度，win/lose 提供目标
  - 建议在 6×6 或 7×7 先验证可行性，再用于 8×8

- [ ] **A9. 7×7×7 继续训练到真正 plateau**：EXP-019b-continue 在 13M 步时 entropy 仍在下降（-0.53），success_rate 仍在上升（45% training），尚未完全 plateau。理论上限可能在 55-65%。了解 7×7×7 的真实天花板对决定是否值得在 8×8×10 投入更多算力有参考价值。
  - 低算力代价（续训 5-10M 步），高信息价值

- [ ] **A10. 8×8×10 超长训练（50M+ 步）**：EXP-019c 在 8M 步后 success_rate 仍为 0-3%。8×8×10 随机完成率约 17%，模型可能需要极大量样本才能首次"撞对"并开始学习。增加到 50M 步（配合更多并行环境 n_envs=32）可能突破探索鸿沟。
  - 高算力代价，结果不确定；建议与 A7（内在动机）结合

## 方向 B：提升训练效率

- [ ] **B1. 最优 checkpoint 自动选择**：EXP-007 的 1.75M 优于 2M final（87% vs 86%），训练完取 final 会损失 1-2%。在训练时加入 eval callback，每隔若干步评估并自动保存最优 checkpoint，对大棋盘长训练尤其重要（避免浪费后期偏移）。

- [ ] **B2. entropy 衰减**（ent_coef 从 0.01 cosine 降到 0.001）：EXP-017 的 entropy 过快收敛（-0.60），EXP-018 停在 -0.70 为正常水平。主动在 late-training 降低 ent_coef，可以在 plateau 后收紧策略、减少无效探索，可能提升 1-2% eval。

- [ ] **B3. 增加并行环境数**：当前 n_envs=16，大棋盘 episode 更长（平均步数 ~3×），提升至 32 或 64 可加速有效样本收集。需评估 T4 GPU 内存和 CPU subproc 瓶颈。

## 方向 C：模型结构（低优先级）

EXP-008 已排除容量假设（features_dim 256 无收益），以下改进独立于容量问题：

- [ ] **C1. 分离 pi/vf 特征提取器**：当前 policy 和 value function 共享 CNN，策略梯度更新干扰 value 拟合（尤其在大棋盘稀疏场景下，EV 爬升缓慢可能部分源于此）。分离后 value head 可独立优化。
- [ ] **C2. 更深 pi/vf 全连接层**（[128,128]）：当前 [64,64]，对 8×8×10 的复杂局面决策可能不足。与 A7 结合验证。
