## 方向 B：更高效的训练（样本效率 / 超参）

目标：用更少步数达到同等或更好的效果，为大棋盘训练节省资源。

- [x] **B1. 学习率 schedule**（cosine 衰减）：lr_start=0.001 → lr_end=0.0001 → **EXP-014 完成，0%**（高 LR 探索无效，稀疏奖励瓶颈非 LR 问题）
- [ ] **B2. 最优 checkpoint 策略**：已验证 1.75M > 2M（87% vs 86%），训练时加入 eval callback 自动选最优
- [ ] **B3. entropy 衰减**（ent_coef 从 0.01 线性降到 0.001）：后期减少无效探索
- [ ] **B4. reward 重新标定（win/lose 幅度缩小）**：当前 win=1.0/lose=-1.0 导致 explained_variance 卡在 0.2~0.45，通过缩小终止信号幅度恢复 value function 可预测性。

  **背景分析**

  win 和 lose 是互斥二选一事件，终止奖励方差 = (win−lose)²·p·(1−p)。
  当前 ±1.0：SD ≤ 1.0；降到 reveal（0.1）量级：SD ≤ 0.1，value 拟合难度下降约 10×。

  EXP-003（win=0.2, lose=−0.05）EV=0.84 → EXP-004（win=1.0, lose=−1.0）EV=0.44：
  唯一改变是终止信号幅度，EV 腰斩，因果关系明确。

  **三个有价值的方向，对应不同设计哲学**：

  1. **win > |lose|（EXP-003 原始）**：死亡惩罚极小（−0.05），赢了才有小奖励（+0.2）。
     扫雷哲学：「不强调避死，鼓励随便探索，胜利是自然涌现的结果。」
     已知有效：EV=0.84，历史最优。副作用未知：agent 可能鲁莽、不避雷。

  2. **|lose| > win（方向反转）**：死亡惩罚大于赢的奖励（如 lose=−0.2, win=0.05）。
     扫雷哲学：「首要任务是活下去；赢只是活到最后的副产品。」
     直觉上符合人类玩扫雷的心理；但 reveal 已经覆盖了「活下去」激励，lose 是否需要更大存疑。

  3. **对称小幅（lose = −win）**：方差最小化配置（如 win=0.1, lose=−0.1）。
     纯工程视角：在给定幅度下对称是最优的，无任何哲学假设，直接对比方差效果。

  **候选实验（均保持 reward_reveal=0.1，在 5×5×3 上验证 EV 后决定是否用于课程学习）**

  - [ ] **B4-1（还原 EXP-003）**：`win=0.2, lose=-0.05`，验证 EV=0.84 能否在当前架构（双通道 obs）复现，以及 eval 胜率是否与 EXP-007（87%）相当
  - [ ] **B4-2（方向反转）**：`win=0.05, lose=-0.2`，测试「强避死 + 弱胜利」是否同样能保持低方差，同时 agent 行为更谨慎
  - [ ] **B4-3（对称极小）**：`win=0.1, lose=-0.1`，最小化终止方差的工程基线，对比前两者是否比不对称更优或更差
